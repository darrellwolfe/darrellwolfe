Assessor Parcels Demo

This is a static parcel dashboard built from local assessor CSV exports and county parcel geometry, with one shared dashboard page for maps, graphs, and tables.

Information Dashboard

Demo Scope

Static files only. No live database connections.

What This Demo Uses

Parcel polygons are derived from the county shapefile and converted into GeoJSON. Parcel attributes and metrics come from the local CSV extracts in assets/data/assessor_data.

How The Dashboard Is Organized

Maps, Graphs, and Tables each live in their own section inside one page.

Each section has its own dropdown so you can swap in new views later without changing the shell around it.

The current modules are parcel polygons by District, parcel counts by District and GEO, and a parcel detail table.

Shared Filters

The unified dashboard uses one shared filter state across all sections. District stays as button chips, GEO uses a multi-select control, and AIN/PIN partial searches feed the same scope everywhere.

Full Dataset Scope

This dashboard now uses the full parcel CSV dataset. The map joins parcel geometry to the parcel records by lrsn, while the graphs and table work directly from the full parcel frame.

Data Notes

The dashboard is static. It does not connect to a live database, and it reads only the locally prepared JSON and GeoJSON files generated from your CSV and shapefile exports.

The live dashboard uses dropdown-selected modules so additional maps, graphs, and tables can slot into the existing layout instead of forcing another page split.

Data Project: Assessor Dashboard

DataViz Assessor's Dashboard is the actual project. The public demo linked below is a sanitized re-presentation built from exported CSV and parcel geometry so outside viewers can explore the workflow without access to the live environment.

Open Public Demo Maps Graphs Tables

Problem

The core project is a real assessor analytics and visualization workflow, not the static demo itself. The work centers on preparing parcel, valuation, and tax-oriented views for dashboard use, then turning those outputs into clear map, graph, and table experiences for exploration and decision support.

Data

  • The project SQL lives in assets/data/assessor_data/*.sql
  • Those queries shape parcel, valuation, category, and net-tax views from the assessor environment
  • The public demo uses sanitized CSV exports generated from those project-side queries
  • County parcel geometry is converted for browser mapping in the public demo

Project vs Demo

The distinction matters here:

  • The project is the DataViz Assessor's Dashboard work itself, including the SQL logic and the original dashboard workflow
  • The demo is a public-facing interactive stand-in built from sanitized exports because site visitors cannot access the original environment
  • The demo is meant to show interaction patterns, dashboard structure, and data storytelling approach without exposing restricted source systems

Approach

The project-side workflow starts with SQL and structured extracts, then the public demo repackages those results for static delivery:

  1. Use assessor SQL queries to assemble parcel, assessed value, category, and net-tax datasets.
  2. Export sanitized CSV outputs from the original dashboard workflow.
  3. Convert those exports into browser-ready static bundles for the public demo.
  4. Present the sanitized outputs through linked maps, graphs, and tables that share filter state.

This is the category lookup table. It pulls active impgroup code records so the dashboard can translate category codes into readable labels when showing assessed values by category.

    --On a.group_code = c.tbl_element
Where c.code_status = 'A'

values_assessed.sql

This query rolls up last year's total assessed value by parcel. It is the simplest valuation measure in the set and gives the dashboard a single assessed-value figure per lrsn.

Select
i.RevObjId AS lrsn,
SUM(c.ValueAmount) AS CadValue_TotalAssessed
FROM CadRoll r
JOIN CadLevel l ON r.Id = l.CadRollId
JOIN CadInv i ON l.Id = i.CadLevelId
JOIN tsbv_cadastre AS c 
  On c.CadRollId = r.Id
  And c.CadInvId = i.Id
  And c.ValueType = 109 -- Variable
WHERE r.AssessmentYear = Year(GetDate()) - 1
GROUP BY i.RevObjId

values_net_tax_value.sql

This query calculates last year's net taxable value by parcel. In the dashboard it supports the comparison between total assessed value and the portion that remains taxable after exemptions and other adjustments.

Select
i.RevObjId AS lrsn,
SUM(c.ValueAmount) AS CadValue_NetTax
FROM CadRoll r
JOIN CadLevel l ON r.Id = l.CadRollId
JOIN CadInv i ON l.Id = i.CadLevelId
JOIN tsbv_cadastre AS c 
  On c.CadRollId = r.Id
  And c.CadInvId = i.Id
  And c.ValueType = 455 -- Variable
WHERE r.AssessmentYear = Year(GetDate()) - 1
GROUP BY i.RevObjId

Repro Steps

  3. Run scripts/assessor_data/build-demo-dataframes.ps1 to convert those exports into demo data bundles.
  4. Keep the static demo files in demos/assessor-dashboard/ and render/sync them into docs/.

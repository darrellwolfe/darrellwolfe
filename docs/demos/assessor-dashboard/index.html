Assessor Parcels Demo

This is a static parcel dashboard built from local assessor CSV exports and county parcel geometry, with separate dashboard pages for maps, graphs, and tables.

Information Maps Graphs Tables

Demo Scope

Static files only. No live database connections.

What This Demo Uses

Parcel polygons are derived from the county shapefile and converted into GeoJSON. Parcel attributes and metrics come from the local CSV extracts in assets/data/assessor_data.

Maps

Parcel polygons are colored by District, filtered by the shared District/GEO/AIN/PIN controls, and AIN/PIN matches stay highlighted in bright yellow.

The page also includes a map view dropdown so more map modules can be added later without changing the layout.

Graphs

Graphs use the same shared filter state as the map and table pages, and summarize parcel counts by District and GEO from the full parcel frame.

The graph view dropdown is where additional chart modules can be slotted in later.

Tables

Tables show the parcel rows behind the current shared filter scope, with paging so the full dataset stays readable instead of trying to render every row at once.

The table view dropdown is ready for more table modules later.

Shared Filters

All dashboard pages share the same filter state through browser storage. District stays as button chips, GEO uses a multi-select control, and AIN/PIN partial searches move with you from page to page.

Full Dataset Scope

This dashboard now uses the full parcel CSV dataset. The map joins parcel geometry to the parcel records by lrsn, while graphs and tables work directly from the full parcel frame.

Data Notes

The dashboard is static. It does not connect to a live database, and it reads only the locally prepared JSON and GeoJSON files generated from your CSV and shapefile exports.

You can reach each viz type either from the top menu on every dashboard page or from the linked descriptions here on the information page.
